# 客戶 Geofeed 提交指南

歡迎！此目錄 (`client_feeds/`) 用於接收並自動化處理您提交的 Geofeed 檔案。

為確保您的 IP 範圍能被正確且及時地更新到主 `geofeed.csv` 檔案中，請您**務必**遵循以下步驟和規範。

## 提交流程

我們使用 GitHub 的 Pull Request (PR) 工作流來接收您的變更。

1.  **Fork 倉庫**：如果您尚未操作過，請先 Fork 本倉庫到您自己的 GitHub 帳號下。
2.  **建立檔案**：在您的 Fork 中，進入 `client_feeds/` 目錄。
3.  **新增/修改您的檔案**：
    * **新客戶**：請在此目錄下建立一個以您公司或組織命名的 `.csv` 檔案（例如：`my_company.csv`）。
    * **現有客戶**：請直接修改您先前建立的那個 `.csv` 檔案。
4.  **重要提示**：**請勿**直接修改位於倉庫根目錄的 `geofeed.csv` 檔案。該檔案是自動生成的。
5.  **提交 PR**：將您的變更（新增或修改的 `.csv` 檔案）提交一個 Pull Request 到本倉庫的 `main` 分支。

## 檔案格式與驗證規則

您的 PR 提交後，系統會自動執行一系列檢查。**如果檢查失敗，您的 PR 將無法被合併。**

請確保您的 `.csv` 檔案嚴格遵守以下規則：

### 1. 檔案格式 (CSV)

檔案必須是標準的 CSV 格式。每一行代表一條 Geofeed 紀錄，至少包含 2 列，最多 4 列：

`IP範圍,國家代碼,[區域代碼],[城市名稱]`

**範例：**
```csv
# 這是一行註解 (自動化腳本會忽略)
2602:f92a:a000::/40,TW,TW-TPE,Taipei,
44.30.30.0/24,JP,JP-13,Tokyo,
